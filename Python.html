Functions

Functions do things. Click this box to learn more.

Keywords

Keywords are the don't-touch-this vocab of Python.

Operators

Operators are a defacto extension of keywords - you'll notice some are the same ("or", "in", "is"...)

Data Types

You need to know what data types you're working with. Well, actually, initially you kinda don't, Python will make a sensible guess for you. But big picture, you really do.

Methods

Methods are like datatype-specific functions, with slightly different syntax.

Modules

Modules are "add-ons" to Python's functionality - they contain more useful functions, and more...

OOP

Python's useful for Object Oriented Programming, or OOP, which a favoured approach to many problems.

Functional Programming

Functional Programming is also possible in Python; having both OOP and Functional makes it particularly flexible.

Syntax

Syntax is important in any language. Python's is one of the most straightforward, but it's still easy to make a mistake. I recommend an IDE, or at least a code editor like Notepad++; they give you a heads-up if something's not right.

Open Source

Python is an Open-Source language. Guido Van Rossum, retired Benevolent Dictator For Life, wanted this programming language to be both intuitive, so that people could use it, and open-source, so that people could make it better. Tres benevolent!

Kittens

Not relevant to Python in the least, but I thought it worth mentioning that kittens are adorable and fantastikitty.

Interpreted

Languages are either Interpreted or Compiled. Python is interpreted, which means that the engine figures out what you want it to do as it does it - easier to work with on-the-hop, but perhaps not as portable (it needs an "interpreter" and these are usually platform-dependent)
